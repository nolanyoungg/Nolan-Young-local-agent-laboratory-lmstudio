# Troubleshooting LM Link

| Symptom                                           | Likely cause                                                                                                                 | Check                                                                                                      | Remediation                                                                                                                                                           |
| ------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Windows cannot see the Mac                        | The devices use different LM Studio accounts, LM Link is disabled, or neither LM Studio nor `llmster` is running on the Mac. | On the Mac, run `lms link status --json`.                                                                  | Sign both devices into the same LM Studio account, enable LM Link, and start LM Studio or `llmster`.                                                                  |
| Link is disconnected                              | The Mac daemon stopped, the Mac slept, or connectivity changed.                                                              | Re-run `lms link status --json` after waking the Mac.                                                      | Restart the Mac daemon or GUI and restore connectivity. The laboratory retries model operations but never changes account or link state.                              |
| Mac is not preferred                              | LM Studio may select an identically keyed Windows model.                                                                     | Inspect the preferred-device setting in LM Link.                                                           | Select the Mac and enable **Set as preferred device**, or run `lms link set-preferred-device` interactively.                                                          |
| Model is absent                                   | The exact model key is not available to the linked Mac.                                                                      | Run `npm run models:lmstudio` on Windows.                                                                  | Download the model on the Mac and set `LM_STUDIO_MODEL` to the returned exact key, not an assumed Hugging Face or Ollama identifier.                                  |
| Windows uses a local duplicate                    | Local and linked variants share a logical key and routing metadata is insufficient.                                          | Run `npm run --silent check:lmlink -- --json` and observe the active device in LM Studio during inference. | Set the Mac as preferred and, if appropriate, manually unload the unwanted Windows instance. A successful response alone is not proof of Mac execution.               |
| API server is unavailable or authentication fails | The Windows server is stopped, authentication is enabled, or the token is invalid.                                           | Run `npm run --silent check:lmstudio -- --json`.                                                           | Start the Windows Developer server or `lms server start`; configure a valid LM Studio API token when required. Never use an OpenAI key or LM Studio account password. |
| Loading or prediction times out                   | A remote cold load is slow, the Mac slept, or memory is constrained.                                                         | Inspect the typed timeout check and LM Studio activity.                                                    | Keep the Mac awake, check memory, and conservatively increase the distinct load or prediction timeout. Retries remain bounded.                                        |
| Remote inference disconnects mid-run              | LM Link or the remote runtime was interrupted.                                                                               | Inspect the sanitized run diagnostics and current link state.                                              | Restore the link and retry. Partial generations are discarded, completed tool calls are retained, and active watchers are terminated on unrecoverable exit.           |
| SDK and LM Studio versions differ                 | The native API or pinned SDK is incompatible with the installed application.                                                 | Run `npm run --silent check:lmstudio -- --json`.                                                           | Upgrade LM Studio first. The repository pins the published SDK and uses native REST for authenticated operation because SDK 1.5.0 lacks constructor token support.    |
